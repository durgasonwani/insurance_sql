# 🗃️ SQL Practice Project


## 📌 Project Overview

This repository contains a complete SQL practice project designed to build strong SQL skills from basic to advanced level.

It includes real-world SQL concepts such as queries, joins, constraints, views, stored procedures, window functions, subqueries, error handling, and triggers.

The project uses the `classicmodels` sample database along with custom tables for practice.

---

## 📚 Topics Covered

### 🔍 1. Basic SQL Queries
Learn basic SQL commands used to retrieve and filter data.

- `SELECT`
- `WHERE`
- `AND`
- `OR`
- `DISTINCT`
- `LIKE`
- Filtering employee, customer, and product records

### 🧩 2. CASE Statements
Use conditional logic to create categories and segments.

- Customer segmentation
- Conditional data classification
- Creating custom labels based on values

### 📊 3. GROUP BY and HAVING
Analyze grouped data using aggregate functions.

- Top products by quantity ordered
- Monthly payment frequency
- Filtering grouped results using `HAVING`

### 🔐 4. SQL Constraints
Practice creating tables with data validation rules.

- Primary Key
- Foreign Key
- Unique
- Not Null
- Check
- Auto Increment
- Sample record insertion and validation

### 🔗 5. SQL Joins
Combine data from multiple tables.

- `INNER JOIN`
- Customer and order analysis
- Top countries by order count
- Product and order details

### 👥 6. Self Join
Analyze employee and manager relationships.

- Employee–Manager relationship
- Reporting hierarchy
- Custom project table analysis

### 🏗️ 7. DDL Commands
Create and modify database objects.

- `CREATE`
- `ALTER`
- `RENAME`
- Table structure modification

### 👁️ 8. SQL Views
Create reusable virtual tables for reporting.

- Product category sales view
- Simplified reporting queries
- Data security and reusability

### ⚙️ 9. Stored Procedures
Create reusable SQL programs.

- Procedures with parameters
- Country-wise payment summary
- Reusable business logic

### 📈 10. Window Functions
Perform advanced data analysis without grouping rows.

- `RANK`
- `DENSE_RANK`
- `LEAD`
- `LAG`
- Year-over-year growth calculation

### 🔄 11. Subqueries
Use nested queries to solve complex problems.

- Products above average price
- Customer and order filtering
- Nested query analysis

### ⚠️ 12. Error Handling
Handle errors safely in stored procedures.

- `HANDLER`
- `ROLLBACK`
- `SIGNAL`
- Exception handling

### 🔁 13. Triggers
Automate actions before or after database events.

- Data validation before insert
- Convert negative working hours into positive values
- Automatic data correction
